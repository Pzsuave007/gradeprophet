The user wants to expand their web app, GradeProphet, into a full-fledged trading platform named FlipSlab Engine. The vision is to create an Operating System for Sports Card Traders.

**PRODUCT REQUIREMENTS:**
1.  **New Platform Structure:** Reorganize the app with a new sidebar: Dashboard, Inventory, Market, Flip Finder, Listings, Account.
2.  **Modular Development:** Build the new modules one by one.
3.  **Core Goal:** Create a central dashboard to track collections, monitor real market values from eBay, detect profitable flips, automate eBay listings, and monitor market price changes.
4.  **Module: Flip Finder**: Preserve all existing GradeProphet functionality (AI grade analysis, eBay monitor) within this module.
5.  **Module: Dashboard**: Provide a main overview displaying total cards, collection value, active listings, market movers, and recently scanned cards.
6.  **Module: Inventory System**: Allow users to manage their card collection with fields for card details, grade, purchase price, and images. Users should be able to add, edit, delete, and search their inventory, categorizing items as 'For Sale' or 'Collection'.
7.  **Module: Market Data**: When a card is opened, the system should query eBay for recent sold listings to calculate market value (median price) and display recent sales.
8.  **eBay Integration**: Connect the user's personal eBay account to pull real data for active listings, sales, and to eventually list items for sale.
9.  **UI/UX**: Implement a toggle to switch between a list view and a more visual grid/card view for all sections displaying collections of cards.
10. **Module: Listings**: Allow users to create listings from inventory, view their active eBay listings, edit them (price, title, etc.), and compare their price against real market data.
