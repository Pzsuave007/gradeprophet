<analysis>**original_problem_statement:**
The user wants to build a web app, GradeProphet, to help them decide which sports cards to send for professional grading by PSA.

**PRODUCT REQUIREMENTS:**
1.  **Core Functionality:** Upload photos of sports cards (front, back, corners) for an AI to analyze based on PSA's grading standards (centering, corners, surface, edges) and predict a grade.
2.  **AI Integration:** Use OpenAI's Vision API. The user provides their own API key.
3.  **PSA 10 Reference Library:** Upload a reference image of a PSA 10 graded version of the same card. The app should read the card details from the PSA slab's label, store the image in a library, and use it for comparison during analysis.
4.  **Analysis History & Feedback Loop:** Keep a history of all analyses and allow the user to input the actual grade received from PSA to create a feedback loop for the AI.
5.  **Card Age Consideration:** The AI must account for the card's age, applying more lenient standards for vintage cards, mimicking real-world PSA grading. This can be detected automatically from a PSA 10 reference, estimated by the AI, or entered manually.
6.  **eBay Integration (New):**
    *   Allow the user to paste an eBay listing URL (both full and short share links).
    *   The system must automatically scrape all images from the listing and display them as thumbnails in the UI.
    *   The user must be able to assign the imported images to the correct slots (Front, Back, Corners).
    *   **Bonus:** If the listing doesn't include corner photos, provide a button to auto-generate corner/edge crops from the main front image.

**User's preferred language**: Spanish. The next agent MUST respond in Spanish only.
